# 수수료 없는 배달 플랫폼 - 초기 조사 보고서

> 작성일: 2026-03-19
> 참고: 2025년 5월까지의 학습 데이터 기반. 최신 변동사항은 별도 검증 필요.

---

## 1. 한국 배달 시장 현황

### 1-1. 시장 규모
| 항목 | 수치 |
|------|------|
| 2023년 거래액 | 약 25~27조 원 |
| 2024년 추정 | 약 28~30조 원 |
| 코로나 이전(2019년) | 약 9~10조 원 |
| 연평균 성장률 | 약 20~25% (2019~2024) |

- 코로나 특수 이후 성장 둔화, 시장 성숙기 진입
- 월평균 약 2~2.5조 원 거래

### 1-2. 주요 플레이어 & 점유율 (2024년 추정)

| 업체 | 점유율 | 소유주 | 외국자본 |
|------|--------|--------|----------|
| **배달의민족** | 60~65% | Delivery Hero SE (독일) | O |
| **쿠팡이츠** | 25~30% | Coupang, Inc. (미국 NYSE) | O |
| **요기요** | 5~8% | GS리테일 컨소시엄 | 부분적 |
| **기타** (땡겨요, 배달특급 등) | 2~5% | 다양 | - |

핵심: 1, 2위 모두 외국 자본. 양사 합산 약 90% 과점 구조.

### 1-3. 수수료 구조

| 플랫폼 | 중개수수료 | 소비자 배달비 | 기사 건당 지급 |
|--------|-----------|-------------|--------------|
| 배달의민족 | 6.8% | 2,000~5,000원 | 3,000~5,500원 |
| 쿠팡이츠 | 9.8% | 0~5,000원 (와우 무료) | 3,500~6,000원 |
| 요기요 | ~12.5% | 2,000~4,000원 | - |
| 땡겨요 | ~2% | 소비자 부담 | - |

### 1-4. 이해당사자별 주요 불만

**점주:**
- 높은 수수료 (6.8~12.5%) + 광고비 부담
- 수수료 체계 일방적 변경
- 플랫폼 종속성, 리뷰 시스템 스트레스

**배달기사:**
- 불안정한 고용 (특수고용직)
- 사고 위험, 배달 단가 하락
- AI 배차 알고리즘 불투명

**소비자:**
- 배달비 지속 인상 (건당 3,000~5,000원+)
- 매장 대비 20~30% 높은 배달 가격
- 구독 피로 (와우, 배민클럽 등)

### 1-5. 규제 동향
- 2023년: 배민 수수료 체계 변경 논란 → 영세 소상공인 2% 우대 도입
- 2024년: 온라인 플랫폼 공정화법 추진, 수수료 투명성 강화
- 플랫폼 노동자 보호법 확대 (산재/고용보험)
- 공공배달앱 확산 후 일부 통폐합 논의

## 2. 수수료 없는/대안 모델 사례

### 2-1. 국내 공공배달앱

| 앱 | 지역 | 수수료 | 상태 | 평가 |
|----|------|--------|------|------|
| **배달특급** | 경기도 | 0% (결제 1~2%) | 가장 성공적, 누적 1조원+ | 경기화폐 연동이 핵심 |
| **먹깨비** | 군산 | 0% | 소규모 지속 | 확장 제한 |
| **띵동** | 대전/충남 | 0% | 이용률 저조 | - |
| 기타 다수 | 각 지자체 | 0% | 대부분 부진 | 통폐합 논의 |

**공공배달앱 공통 실패 원인:**
1. 3면 플랫폼의 냉기시동 문제 (가맹점-소비자-라이더 동시 확보 어려움)
2. 민간 대비 현저히 낮은 기술 수준 (UX, 알고리즘)
3. 마케팅 예산 한계
4. 배달 라이더 인프라 부재
5. 지자체 예산 의존 → 정치적 변동에 취약
6. 소비자 전환 동기 부족 (이미 배민/쿠팡이츠에 익숙)

### 2-2. 해외 사례

| 모델 | 사례 | 핵심 | 평가 |
|------|------|------|------|
| 협동조합 | CoopCycle (유럽) | 오픈소스, 라이더 소유 | 소규모, 규모 확장 어려움 |
| 협동조합 | Mensakas (스페인) | 라이더 직접 설립 | 공정 노동, 규모 제한 |
| 오픈 프로토콜 | **ONDC (인도)** | 정부 오픈 프로토콜 | **가장 유망한 대안** |
| 수수료 상한 규제 | 뉴욕시 (미국) | 수수료 15% 상한 | 기존 인프라 활용 |
| SaaS/구독 | ChowNow (미국) | 월 구독료, 수수료 0% | 배달 인프라 별도 필요 |

### 2-3. 대안적 수익 모델 옵션

| 모델 | 설명 | 실현 가능성 |
|------|------|-----------|
| **광고** | 앱 내 검색/배너 광고 | 트래픽 확보 후 가능 |
| **소비자 구독** | 월정액 배달비 무료 멤버십 | 중기 (사용자 기반 확보 후) |
| **점주 구독** | 월정액 이용료 (수수료 대체) | 초기부터 가능 |
| **데이터** | 상권 분석, 소비 트렌드 리포트 | 장기 |
| **금융 서비스** | 점주 대출, 선정산 | 중장기 |
| **공유 주방** | 클라우드 키친 임대 | 중장기, 높은 초기 투자 |
| **물류 확장** | 비음식 배달 인프라 공유 | 장기 |
| **공공 보조금** | 지자체/정부 지원 | 초기 부트스트랩 |

### 2-4. 핵심 교훈

1. **수수료 0%만으로는 경쟁 불가** - 소비자는 가맹점 다양성, 배달 속도, UX로 선택
2. **3면 플랫폼 냉기시동** - 한 쪽에 강력한 인센티브 집중 투입으로 임계점 돌파 필요
3. **기술 투자 필수** - 오픈소스 활용(CoopCycle)이 현실적
4. **ONDC 모델 주목** - 개방형 프로토콜이 독점 대안으로 가장 유망
5. **수익 다각화 필수** - 광고 + 구독 + 데이터 + 금융 복합 모델
6. **지속가능성 설계** - 100% 보조금 의존은 위험, 자체 수익 구조 병행

## 3. 기술 아키텍처

### 3-1. 필요 시스템

| 시스템 | 핵심 기능 |
|--------|----------|
| **소비자 앱** | 주소 기반 가게 탐색, 주문/결제, 실시간 추적, 리뷰 |
| **점주 앱/웹** | 메뉴 관리, 주문 수신/처리, 정산 조회 |
| **배달기사 앱** | 배차 수신, 네비게이션, 수익 관리 |
| **관리자 웹** | 가게 심사, 주문 모니터링, 정산, 대시보드 |

### 3-2. 추천 MVP 기술 스택

```
모바일:       Flutter (소비자 + 배달기사 앱)
점주/관리자:  React + TypeScript (웹)
백엔드:       NestJS (TypeScript) - 모놀리식
DB:           PostgreSQL (PostGIS) + Redis
결제:         포트원 + 토스페이먼츠
지도:         카카오맵 API + Tmap API
실시간:       Firebase (FCM + Realtime DB)
파일저장:     AWS S3
인프라:       AWS (EC2/ECS) 또는 Railway/Supabase
CI/CD:       GitHub Actions
모니터링:     Sentry + CloudWatch
```

### 3-3. 아키텍처 개요

```
[소비자 앱]  [점주 웹]  [기사 앱]  [관리자 웹]
     |           |          |           |
     +-----------+----------+-----------+
                 |
         [API Gateway / LB]
                 |
         [NestJS 백엔드 (모놀리식)]
          /    |     |      \
     [Auth] [Order] [Store] [Delivery]  ← 모듈 단위 분리
          \    |     |      /
           +---+-----+---+
           |             |
      [PostgreSQL]   [Redis]
       (PostGIS)    (캐시/큐/위치)
           |
      [Firebase]     [S3]
     (실시간/푸시)  (이미지)
           |
      [포트원 → 토스페이먼츠]
```

### 3-4. MVP 개발 로드맵

**Phase 1 - 핵심 MVP (3~4개월)**
- 소비자: 소셜 로그인, 주소 설정, 가게/메뉴 탐색, 주문, 결제, 주문 상태 확인
- 점주: 가게/메뉴 등록, 주문 수신/처리
- 기사: 배달 요청 수신, 픽업/완료 처리
- 관리자: 가게 승인, 주문 모니터링, 기본 정산

**Phase 2 - 성장 기능 (2~3개월)**
- 실시간 기사 위치 추적, 자동 배차, 리뷰 시스템
- 쿠폰/프로모션, 간편결제, 푸시 알림
- 점주 매출 대시보드, 기사 수익 현황

**Phase 3 - 고도화 (6개월+)**
- AI 배차 최적화, 묶음 배달, 예약 주문
- 구독 서비스, 분석 대시보드, 채팅

### 3-5. 인프라 비용

| 단계 | 일 주문량 | 월 비용 |
|------|----------|---------|
| MVP | 100~500건 | 약 20~50만원 |
| 성장 | 5,000~50,000건 | 약 150~500만원 |

- AWS Activate 스타트업 크레딧 ($10,000) 활용 가능
- 네이버 클라우드 스타트업 프로그램도 검토 대상
