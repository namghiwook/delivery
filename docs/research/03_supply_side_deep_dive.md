## 1. 점주의 실제 수익 구조

### 1-1. 배달 음식점의 매출 대비 비용 구조

일반적인 배달 전문 음식점(월 매출 3,000만원 기준)의 비용 구조는 다음과 같다.

| 항목 | 비율 | 금액(월) | 비고 |
|------|------|----------|------|
| **원재료비** | 30~35% | 900~1,050만원 | 업종별 차이 큼. 치킨 25~30%, 중식 30~35%, 분식 20~25% |
| **인건비** | 20~25% | 600~750만원 | 사장 본인 인건비 포함 시. 1인 운영이면 0~10% |
| **임대료** | 8~12% | 240~360만원 | 배달 전문점은 상대적 저렴(비목 위치 가능). 서울 평균 150~300만원 |
| **플랫폼 수수료** | 6.8~12.5% | 204~375만원 | 배민 6.8%, 쿠팡이츠 9.8%, 요기요 ~12.5% |
| **배달대행 수수료** | 8~15% | 240~450만원 | 건당 3,000~4,500원. 점주 부담분(소비자 배달비와 별도) |
| **광고비(플랫폼)** | 3~10% | 90~300만원 | 배민 울트라콜 월 8.8만원/건, 오픈리스트 등 |
| **포장재/소모품** | 2~3% | 60~90만원 | 용기, 비닐, 수저 등 |
| **공과금/기타** | 3~5% | 90~150만원 | 전기, 가스, 수도, 보험, 카드수수료(1.5~2.5%) |
| **부가세/세금** | 실효 3~5% | 90~150만원 | 간이과세 해당 시 더 낮음 |
| **합계 비용** | **84~112%** | **2,514~3,675만원** | |
| **순이익** | **-12% ~ +16%** | **-675 ~ +486만원** | |

**핵심 수치:**
- 배달 음식점의 평균 순이익률: **약 5~10%** (잘 운영되는 경우)
- 많은 영세 음식점의 실질 순이익률: **0~5%** (사장 본인 인건비 제외 시)
- 사장 본인 인건비를 포함하면 적자인 경우도 상당수

### 1-2. 비용 항목별 세부 분석

**플랫폼 비용의 실체 (수수료 + 광고비 + 배달대행비)**

점주 입장에서 "플랫폼에 나가는 돈"은 중개수수료만이 아니다. 실제로는 3중 구조:

| 비용 항목 | 월 매출 3,000만원 기준 | 누가 가져가나 |
|----------|---------------------|-------------|
| 중개수수료 (배민 6.8%) | 204만원 | 배민(우아한형제들) |
| 광고비 (울트라콜 2개 + 오픈리스트) | 약 25~80만원 | 배민 |
| 배달대행 수수료 (건당 점주부담분) | 약 150~300만원 | 배달대행사(부릉/바로고 등) |
| **합계** | **약 379~584만원** | |
| **매출 대비 비율** | **12.6~19.5%** | |

즉, 점주가 체감하는 "플랫폼 관련 총비용"은 수수료 6.8%가 아니라 **매출의 13~20%**에 달한다.

### 1-3. 수수료 6.8% 감소가 점주 순이익에 미치는 실제 영향

**시나리오: 배민 수수료 6.8% → 우리 플랫폼 0%**

| 항목 | 배민 이용 시 | 우리 플랫폼(수수료 0%) | 차이 |
|------|-----------|-------------------|------|
| 월 매출 | 3,000만원 | 3,000만원 | - |
| 중개수수료 | 204만원 (6.8%) | 0원 | **-204만원** |
| 기타 비용 동일 가정 | 2,310만원 | 2,310만원 | - |
| **순이익** | **486만원** | **690만원** | **+204만원** |
| **순이익률** | **16.2%** | **23.0%** | **+6.8%p** |

순이익률 기준으로 보면:
- 순이익률 10%인 점주: 수수료 6.8% 절감 시 순이익이 **68% 증가** (10% → 16.8%)
- 순이익률 5%인 점주: 수수료 6.8% 절감 시 순이익이 **136% 증가** (5% → 11.8%)
- 손익분기 수준 점주: 수수료 절감분이 곧바로 순이익 = **생존과 폐업의 차이**

**이것이 핵심 셀링포인트다.** 수수료 6.8%는 매출 대비로는 작아 보이지만, 순이익 대비로는 50~130% 수준의 변화를 만든다.

### 1-4. 점주가 "수수료 3%면 충분히 매력적"이라고 느끼는 임계점

점주의 수수료 민감도를 단계적으로 분석한다.

| 수수료율 | 월 매출 3,000만원 기준 수수료 | 점주 반응 예측 | 근거 |
|---------|-------------------------|-------------|------|
| **0%** | 0원 | "당연히 좋지, 근데 진짜?" → 의심 | 지속가능성 의문, "나중에 올릴 것" 우려 |
| **1~2%** | 30~60만원 | "파격적이다" → 강한 전환 동기 | 땡겨요(~2%)가 이 구간. 확실한 차별점 인식 |
| **3%** | 90만원 | "충분히 매력적" → 전환 고려 | 배민 대비 절반 이하. 체감 차이 분명 |
| **5%** | 150만원 | "좀 낫긴 한데..." → 약한 전환 동기 | 배민과 차이가 절대금액 기준 월 54만원. 전환 비용 대비 애매 |
| **6.8%** | 204만원 | "배민이랑 같잖아" → 전환 무의미 | 동일 수수료면 트래픽 많은 배민이 유리 |

**임계점 분석:**
- **0~2%**: "반드시 써봐야 한다" 구간. 적극적 전환 동기 발생
- **3%**: **심리적 임계점**. 배민 6.8%의 절반 이하라는 인식이 중요
- **4~5%**: "조금 싸긴 한데 굳이?" 구간. 다른 혜택(광고비 무료, 더 좋은 UX 등)이 있어야 전환
- **6%+**: 전환 동기 거의 없음

**권장사항:** 점주 대상 커뮤니케이션 시 "배민 6.8%의 절반도 안 되는" 비교 프레이밍이 효과적. 수수료 **0~2%** 구간이 확실한 전환 동기를 만들고, **3%**가 최대 허용선이다.

## 2. 점주의 멀티호밍(다중 입점) 행태

### 2-1. 현재 다중 입점 현황

| 입점 유형 | 추정 비율 | 특징 |
|----------|----------|------|
| 배민 단독 | 10~15% | 주로 배민 충성 또는 소규모 점포 |
| 배민 + 쿠팡이츠 | 35~40% | 가장 일반적인 조합 |
| 배민 + 쿠팡이츠 + 요기요 | 25~30% | 매출 극대화 추구 |
| 배민 + 쿠팡이츠 + 기타(땡겨요/배달특급 등) | 5~10% | 수수료 절감 의지가 강한 점주 |
| 3개 이상 플랫폼 동시 | 30~40% | 멀티호밍 일반화 |

**핵심 인사이트:**
- 배민 미입점 점포는 극히 드물다 (전체의 5% 미만 추정)
- 쿠팡이츠 입점률이 급속 상승 중 (2023년 대비 2024년 약 30%+ 성장)
- 멀티호밍은 이미 보편적 현상이며, **새 플랫폼 추가 입점의 심리적 장벽은 낮다**
- 다만, "관리 가능한 플랫폼 수"에는 한계가 있음 (3~4개가 현실적 한계)

### 2-2. 새 플랫폼 추가 입점의 장벽

| 장벽 요소 | 심각도 | 구체적 내용 | 해결 방안 |
|----------|--------|-----------|----------|
| **태블릿 추가** | 중 | 플랫폼별 전용 태블릿/앱 필요. 주방 공간에 3~4대 이미 있음 | 기존 태블릿/스마트폰 호환. 별도 기기 불필요하게 설계. 또는 통합 솔루션(예: 오더히어로 등 연동) |
| **메뉴 등록** | 중~상 | 메뉴 사진, 설명, 가격, 옵션 일일이 재등록. 50개 메뉴면 반나절 소요 | **타 플랫폼 메뉴 자동 복사 기능** 제공 (배민/쿠팡이츠 메뉴 크롤링 또는 엑셀 업로드) |
| **학습 비용** | 하~중 | 새 앱/대시보드 사용법 익히기. 정산 구조 이해 | 직관적 UX + 온보딩 튜토리얼 + 초기 1:1 전화 지원 |
| **정산 주기/방식 차이** | 하 | 플랫폼마다 정산일, 수수료 계산법 상이 | 명확하고 투명한 정산 구조. D+3 빠른 정산 기본 제공 |
| **주문 분산 우려** | 중 | "새 플랫폼에 올려도 주문이 안 올 텐데" | 초기 3개월 주문 보장 프로그램 또는 주문 없으면 비용 없음(수수료 0%의 장점) |
| **리뷰 재축적** | 상 | 배민에 쌓인 리뷰 500개를 새 플랫폼에서 0부터 시작 | 초기 리뷰 부스팅 프로그램 (첫 리뷰 작성 시 소비자 쿠폰) |
| **배달 커버리지 불안** | 중 | "이 앱으로 주문 오면 배달기사가 잡힐까?" | 배달대행사 연동으로 기존과 동일한 배달 인프라 사용 보장 |

**입점 장벽 종합 평가:**
- 점주가 새 플랫폼에 추가 입점하는 데 드는 실질 비용: **시간 2~4시간 + 학습 1~2일**
- 수수료 0%인 플랫폼에 추가 입점하는 것은 **리스크가 거의 없다** (주문 안 오면 비용도 0원)
- 핵심 장벽은 "시간과 귀찮음"이지 "비용"이 아님
- **메뉴 자동 등록 + 초기 전담 지원**이 장벽을 획기적으로 낮출 수 있음

### 2-3. 점주가 특정 플랫폼을 "메인"으로 쓰게 만드는 요소

단순 입점과 "메인 플랫폼"은 다르다. 점주가 특정 플랫폼을 메인으로 쓴다는 것은:
- 해당 플랫폼의 주문을 우선 처리
- 해당 플랫폼에 광고비를 집중
- 메뉴/가격 업데이트를 해당 플랫폼에서 먼저 진행
- 소비자에게 해당 플랫폼 주문을 유도

| 메인 결정 요소 | 중요도 | 현재 상황 | 우리 플랫폼 전략 |
|--------------|--------|----------|---------------|
| **주문량(트래픽)** | ★★★★★ | 배민 압도적 1위 | 초기 열위. 지역 집중 + 점주 자발적 홍보로 보완 |
| **수수료율** | ★★★★☆ | 배민 6.8%가 기준 | **0~2%로 압도적 차별화** |
| **정산 속도** | ★★★☆☆ | 배민 D+7~14 | D+1~3 빠른 정산으로 차별화 |
| **광고비 대비 효율** | ★★★★☆ | 배민 광고비 높지만 효과도 높음 | 광고비 없이 공정 노출로 차별화 |
| **배달 품질/속도** | ★★★☆☆ | 쿠팡이츠 단건배달이 우수 | 배달대행사 연동으로 동등 수준 확보 |
| **CS/분쟁 처리** | ★★★☆☆ | 대형 플랫폼은 소비자 편향적 | 점주 친화적 CS 정책으로 차별화 |
| **점주 커뮤니티/소속감** | ★★☆☆☆ | 기존 플랫폼에는 없음 | **점주 커뮤니티 구축으로 차별화** |
| **데이터/인사이트** | ★★☆☆☆ | 제한적 제공 | 매출분석, 경쟁 분석 등 무료 제공 |

**"메인 플랫폼" 전환을 위한 필요조건:**

아무리 수수료가 낮아도, 주문이 안 오면 메인이 될 수 없다. 따라서:

1. **최소 주문량 보장**: 해당 지역에서 일 일정 건수 이상의 주문이 발생해야 함
2. **수수료 차이의 체감**: 월 50만원 이상 절감 효과가 있어야 "메인 전환"을 고려
3. **운영 편의성 동등 이상**: 배민보다 불편하면 절대 메인이 안 됨
4. **점주 자율성**: 가격 설정, 영업시간, 메뉴 구성에 대한 자유도

## 3. 점주가 적극적으로 우리 플랫폼을 홍보하게 만드는 방법

### 3-1. 점주 자발적 홍보의 메커니즘

점주가 소비자에게 "이 앱으로 주문해주세요"라고 말하는 것은 매우 강력한 마케팅이다. 이유:

- **신뢰도**: 단골 손님이 가게 사장님의 추천을 신뢰
- **전환율**: 일반 광고 대비 10배+ 높은 전환율
- **비용**: 마케팅 비용 0원
- **지속성**: 한 번 전환된 소비자는 해당 가게 재주문 시 계속 우리 앱 사용

**점주가 홍보하는 실제 시나리오:**

```
시나리오 1 - 매장 내 안내
"저희 가게는 'OO배달'로 주문하시면 2,000원 더 저렴합니다."
→ 포장/매장 주문 손님에게 전단지 또는 스티커 제공

시나리오 2 - 배달 주문 동봉
배달 시 전단지 동봉: "다음 주문은 'OO배달'에서! 첫 주문 3,000원 할인"
→ 모든 배달(배민/쿠팡이츠 포함) 주문에 동봉

시나리오 3 - SNS 홍보
점주의 인스타그램/네이버 블로그:
"수수료가 없어서 더 좋은 재료로 만들 수 있어요. OO배달에서 주문해주세요!"

시나리오 4 - 가격 차별화
배민: 짜장면 8,000원 / 우리 플랫폼: 짜장면 7,000원
→ 수수료 절감분을 소비자 가격 인하로 전가
```

### 3-2. 점주를 영업사원으로 만드는 인센티브 구조

| 인센티브 | 구체적 설계 | 점주 반응 예측 | 비용 |
|---------|-----------|-------------|------|
| **추천 소비자 보상** | 점주 전용 추천 코드 발급. 신규 소비자가 코드로 가입+첫 주문 시 점주에게 2,000원, 소비자에게 3,000원 할인 | 높은 참여 의지 (직접적 금전 보상) | 건당 5,000원 (CAC) |
| **가격 인하 자동 표시** | 점주가 우리 플랫폼에서 더 낮은 가격 설정 시, 앱에 "이 가게는 다른 앱보다 OOO원 저렴" 자동 표시 | 가격 인하 동기 부여 | 0원 |
| **리뷰 인센티브** | 우리 앱 리뷰 작성 소비자에게 500원 쿠폰 → 점주에게도 리뷰 1건당 200원 크레딧 | 리뷰 축적 가속화 | 건당 700원 |
| **월간 우수 점주 선정** | 주문량/평점/성장률 기준 우수 점주 선정 → 앱 내 특별 배지 + 홈 노출 | 자존감 + 실질 혜택 | 소프트 비용만 |
| **점주 추천 점주** | 기존 점주가 신규 점주 추천 시 양측에 월정액 1개월 무료 | 점주 네트워크 활용한 바이럴 | 월정액 1개월분 (5~10만원) |
| **매출 성장 보너스** | 우리 플랫폼 월 매출이 전월 대비 20%+ 성장 시 월정액 50% 할인 | 플랫폼 활용 적극화 동기 | 월정액의 50% |
| **전용 포장재 무료 제공** | "OO배달로 주문하세요" 인쇄된 배달 봉투/스티커 무료 배포 | 자연스러운 브랜드 노출 | 건당 50~100원 |

### 3-3. 가장 효과적인 점주 홍보 전략: "가격 차별화 자동 표시"

점주에게 가장 강력한 홍보 동기는 **"우리 가게 매출이 늘어난다"**이다.

**메커니즘 설계:**

```
1. 점주가 우리 플랫폼에서 메뉴 가격을 배민보다 5~10% 낮게 설정
   (수수료 절감분의 일부를 가격 인하로 전가)

2. 우리 앱에서 해당 가게에 "이 가게 특가" 또는 "앱 전용 할인가" 표시

3. 점주는 배달 시 전단지 동봉:
   "OO배달에서 주문하면 짜장면 1,000원 할인!"

4. 소비자 유입 → 점주 매출 증가 → 점주가 더 적극적으로 홍보
   → 소비자 추가 유입 (선순환)
```

**수치 시나리오:**
- 점주 A의 배민 월 매출: 500만원 (수수료 34만원)
- 우리 플랫폼에서 메뉴 가격 5% 인하 → 소비자에게 매력적
- 우리 플랫폼 월 매출 200만원 달성 시: 수수료 0원, 가격 인하분 10만원
- 점주 순이익 차이: 배민 매출 500만원(수수료 34만원) vs 배민 300만원(수수료 20.4만원) + 우리 200만원(수수료 0원) → **순이익 13.6만원 증가 + 가격 인하분 6만원 = 순 7.6만원 이익**
- 장기적으로 우리 플랫폼 비중이 커질수록 이익 증가 가속

### 3-4. 점주 커뮤니티/네트워크 효과

| 커뮤니티 전략 | 구체적 실행 | 기대 효과 |
|-------------|-----------|----------|
| **점주 전용 온라인 커뮤니티** | 앱 내 또는 카카오톡 오픈채팅방. 지역별 점주 모임 | 정보 교류, 소속감, 이탈 방지 |
| **월간 점주 모임 (오프라인)** | 지역별 월 1회 점주 간담회. 간단한 식사 제공 | 직접적 관계 구축, 피드백 수집 |
| **점주 성공 사례 공유** | "A 사장님, 우리 플랫폼으로 월 200만원 추가 매출!" 콘텐츠 | 신규 점주 전환 유도, 기존 점주 동기부여 |
| **점주 공동 프로모션** | "이 동네 맛집 5곳 묶음 할인" → 점주 협업 이벤트 | 소비자에게 다양성 제공, 점주 간 유대감 |
| **점주 자문단/운영위원회** | 월 1회 점주 대표 5~10명과 정책 논의 | 점주 의견 반영 → "우리 플랫폼" 인식, 플랫폼 종속이 아닌 파트너 관계 |
| **공동 구매 네트워크** | 식자재/포장재 공동 구매 → 비용 절감 | 추가적 경제적 이익, 록인 효과 |

**핵심 원칙: 점주를 "입점 업체"가 아닌 "파트너/공동 운영자"로 대우하는 것**

기존 플랫폼(배민, 쿠팡이츠)은 점주를 "공급자"로만 취급한다. 수수료 체계 일방 변경, 리뷰 정책 일방 적용, CS 분쟁에서 소비자 편향 등이 점주 불만의 근원이다. 우리 플랫폼이 점주를 "공동 운영자"로 대우하면, 이것 자체가 강력한 차별화이자 록인 요소가 된다.

## 5. 라이더의 실제 불만과 니즈

### 5-1. 건당 수입 외에 라이더가 중요하게 여기는 것

라이더 만족도 조사(노동연구원/플랫폼노동 연구 등)에서 반복적으로 나타나는 요소를 중요도 순으로 정리한다.

| 순위 | 요소 | 중요도 | 구체적 내용 |
|------|------|--------|-----------|
| 1 | **콜 안정성 (물량)** | ★★★★★ | "한 시간에 몇 건 잡히느냐"가 시간당 수입을 결정. 건당 단가보다 더 중요한 경우 많음 |
| 2 | **건당 단가** | ★★★★★ | 최소 3,500원 이상이어야 움직이는 동기. 3,000원 이하 콜은 기피 |
| 3 | **배차 공정성/투명성** | ★★★★☆ | "왜 나한테는 먼 거리 콜만 오는가", "신규 라이더에게 좋은 콜 몰아주기" 의심 |
| 4 | **안전/사고 대응** | ★★★★☆ | 사고 시 보험 처리, 치료비 지원, 배달 중단 시 수입 보전 |
| 5 | **정산 속도** | ★★★☆☆ | 빠른 정산 선호 (일 정산 > 주 정산 > 월 정산) |
| 6 | **앱 UX/네비게이션** | ★★★☆☆ | 목적지 정확성, 배달 동선 최적화, 앱 안정성 |
| 7 | **고객/점주와의 분쟁 처리** | ★★★☆☆ | 배달 완료 후 고객 클레임 시 라이더 보호 |
| 8 | **시간 자유도** | ★★★☆☆ | 원하는 시간에 시작/종료할 수 있는 유연성 |
| 9 | **동료 커뮤니티** | ★★☆☆☆ | 같은 지역 라이더 간 정보 교류 (위험 도로, 맛집 위치 등) |
| 10 | **존중/인정** | ★★☆☆☆ | "배달 기사"에 대한 사회적 인식. 플랫폼이 라이더를 존중하는 태도 |

### 5-2. 배차 알고리즘에 대한 구체적 불만

라이더들이 배차 알고리즘에 대해 가장 많이 제기하는 불만:

| 불만 유형 | 구체적 내용 | 빈도 |
|----------|-----------|------|
| **"블랙박스" 불투명성** | 왜 이 콜이 나에게 왔는지, 배차 기준이 뭔지 모름. "AI가 나를 차별하는 것 같다" | 매우 높음 |
| **먼 거리 콜 강제** | 픽업 지점까지 2~3km인 콜 배정 → 이동 시간 대비 수입 낮음 | 높음 |
| **콜 거부 페널티** | 콜을 거부하면 이후 좋은 콜이 안 잡힘 (의심). 실제 패널티 존재 여부 불투명 | 높음 |
| **묶음배달 강제** | 2~3건 묶음 배달 시 건당 단가 하락. 본인 의사와 무관한 묶음 배정 | 중~높음 |
| **피크 시간 쏠림** | 피크에만 콜 집중, 비피크에는 대기 시간 과다 | 중간 |
| **신규 라이더 우대 의심** | 신규 가입 라이더에게 좋은 콜 몰아주기 → 기존 라이더 불이익 | 중간 |
| **평가 기반 차별** | 고객 평점 낮으면 콜 배정 불이익 → 근거 불분명한 평가 기준 | 중간 |
| **동적 단가 불만** | 비피크 시간에 단가가 너무 낮음 (2,500~3,000원). "기름값도 안 된다" | 중간 |

**라이더가 원하는 이상적 배차:**
1. 배차 기준을 명확하게 공개 (거리, 대기시간, 평가 등의 가중치)
2. 콜 거부에 대한 과도한 페널티 없음
3. 묶음배달 선택권 (본인이 묶음 수락 여부 결정)
4. 최소 건당 단가 보장 (예: 어떤 콜이든 최소 3,500원)
5. 배달 동선 최적화 (가까운 콜 위주 배정)

### 5-3. 산재보험/고용안정에 대한 실제 니즈 수준

| 구분 | 전업 라이더 니즈 | 부업 라이더 니즈 |
|------|--------------|--------------|
| **산재보험** | ★★★★★ 절실 | ★★★☆☆ 있으면 좋음 |
| **고용보험 (실업급여)** | ★★★★☆ 원하지만 보험료 부담 우려 | ★★☆☆☆ 관심 낮음 |
| **국민연금** | ★★★☆☆ 미래 대비 원하지만 현재 부담 | ★★☆☆☆ 본업에서 가입 |
| **건강보험** | ★★★★☆ 지역가입자 보험료 부담 큼 | ★★☆☆☆ 직장가입자 또는 피부양자 |
| **유급 휴가/휴일** | ★★★★☆ 원하지만 비현실적이라 인식 | ★☆☆☆☆ 관심 없음 |
| **퇴직금** | ★★★☆☆ 장기 종사자는 원함 | ★☆☆☆☆ 관심 없음 |

**현재 제도 현황 (2024~2025년 기준):**
- 산재보험: 플랫폼 종사자 의무 적용 확대 (배달/택시/대리 등). 비용은 플랫폼/대행사와 라이더가 분담
- 고용보험: 2022년부터 단계적 적용. 보험료 부담에 대한 라이더 반발 존재
- 4대 보험 전체: 특수고용직 적용 확대 추세이나, "근로자성" 논란 지속

**우리 플랫폼에 대한 시사점:**
- 산재보험은 법적 의무이므로 당연 적용. 추가로 **상해보험 무료 제공**이면 차별화
- "고용"이 아닌 "파트너십" 프레이밍 유지하되, 실질적 안전망 제공
- 전업 라이더에게는 "안정적 수입 + 보험"이 매력 포인트
- 부업 라이더에게는 "유연한 시간 + 적정 단가"가 매력 포인트

### 5-4. "콜비 경쟁 없는 플랫폼"에 대한 라이더 반응 예측

**콜비 경쟁이란?**
배달대행사의 전통적 배차 방식. 주문이 들어오면 라이더들이 경쟁적으로 수락 버튼을 누르는 방식. 선착순이라 "빠른 손가락"이 유리하고, 좋은 콜(고단가/가까운 거리)에 라이더가 몰린다.

**콜비 경쟁의 문제점:**
- 앱을 계속 주시해야 하는 스트레스 (운전 중에도)
- "좋은 콜"은 0.5초 만에 사라짐 → 안전 위험
- 경력/숙련도와 무관하게 반응속도로 결정
- 비피크 시간에는 콜 자체가 적어 대기 시간 과다

**"AI 자동 배차 (콜비 경쟁 없음)" 플랫폼에 대한 반응 예측:**

| 라이더 유형 | 예상 반응 | 근거 |
|-----------|----------|------|
| 전업/숙련 라이더 | **경계~부정적** | "좋은 콜을 골라 잡는 것이 내 경쟁력인데, 자동 배차면 그 능력이 무의미해진다" |
| 신규/초보 라이더 | **긍정적** | "선착순 경쟁에서 항상 지니까, 자동 배차가 공정하게 느껴진다" |
| 부업 라이더 | **긍정적** | "앱 계속 쳐다보지 않아도 되니까 편하다" |
| 안전 중시 라이더 | **긍정적** | "운전 중에 콜 잡으려고 핸드폰 보는 게 위험했다" |

**핵심 인사이트:**
- AI 자동 배차는 "공정한 분배"에 대한 니즈와 부합하지만, **숙련 라이더의 반발**이 예상됨
- 해결 방안: **배차 알고리즘의 투명한 기준 공개** + **경력/평가가 높은 라이더에게 소폭의 우선 배차 혜택**
- 쿠팡이츠의 자동 배차가 시장에서 수용되고 있으므로, 트렌드 자체는 자동 배차 방향
- **배민커넥트도 자동 배차**, 대행사만 콜비 경쟁 방식이 잔존

```
Phase 1 (0~12개월): 배달대행사 100% 연동
  → 자체 라이더 0명. 물류 리스크 제거. 서비스 안정성 확보.

Phase 3 (24개월+): 대행사 40% + 자체 라이더 60%
  → 자체 라이더 비중 확대로 배달비 마진 확보
  → AI 배차/묶음배달로 효율 극대화
  → 라이더 복지(보험, 교육, 커뮤니티)로 록인
```

**핵심 이탈 방지 공식:**

```

→ 4가지 모두 경쟁사와 동등하면 "추가 혜택"이 결정 요소

```

## 7. 결론: 점주와 라이더를 동시에 만족시키는 구조 제안

### 7-1. 점주-라이더-플랫폼 간 수익 분배 최적 구조

배달 1건(주문 금액 20,000원, 소비자 배달비 3,000원)의 금액 흐름:

**현재 (배민 기준):**

```
소비자 지불: 20,000원(음식) + 3,000원(배달비) = 23,000원

→ 점주 수취: 20,000 - 1,360(수수료 6.8%) - 3,500(배달대행비 점주부담분) = 15,140원
→ 배민 수취: 1,360원 (수수료)
→ 배달대행사 수취: 3,500원 (소비자 배달비 3,000 + 점주 부담 500원)
  → 이 중 라이더 지급: ~3,000원, 대행사 마진: ~500원
→ PG사: 약 500원 (카드수수료 약 2.5%)

점주 실질 수취율: 75.7% (매출 대비)
```

**우리 플랫폼 제안 구조:**

```
소비자 지불: 19,000원(음식, 5% 할인) + 3,000원(배달비) = 22,000원

→ 점주 수취: 19,000 - 0(수수료 0%) - 0(배달비 점주 부담 없음) = 19,000원
  (단, 월정액 5만원 별도. 일 10건 기준 건당 약 167원)
→ 우리 플랫폼 수취: 월정액 167원/건 + 배달비 마진 200원/건 = 367원/건
→ 배달대행사: 3,000원 (소비자 배달비에서 직접 지급)
  → 라이더: ~2,600원, 대행사 마진: ~400원
→ PG사: 약 475원

점주 실질 수취율: 98.3% (음식 매출 대비, 월정액 차감 전)
                  97.4% (월정액 차감 후)
```

**비교 요약:**

| 항목 | 배민 | 우리 플랫폼 | 차이 |
|------|------|-----------|------|
| 점주 실질 수취율 | 75.7% | 97.4% | **+21.7%p** |
| 소비자 음식 가격 | 20,000원 | 19,000원 | **-5% 저렴** |
| 소비자 총 지불 | 23,000원 | 22,000원 | **-4.3% 저렴** |
| 라이더 건당 수입 | ~3,000원 | ~2,600원(대행사) / ~3,500원(자체) | 대행사 동등 / 자체 시 +17% |
| 플랫폼 건당 수익 | 1,360원 | 367원 | -73% (다른 수익원으로 보전) |

### 7-2. 현실적인 수수료율과 라이더 지급 구조

**점주 대상 비용 구조 제안:**

| 항목 | 구조 | 점주 체감 |
|------|------|----------|
| **중개수수료** | **0%** | "수수료 무료" 핵심 메시지 |

| **결제수수료** | 실비 전가 (약 2~2.5%) | "카드사 수수료니까 어쩔 수 없지" |
| **광고비** | 선택 사항 (월 0~30만원) | 자율적 지출 |
| **배달비 점주 부담** | **0원** (소비자 배달비로 충당) | 큰 메리트. 배민은 점주 부담분 존재 |

### 7-3. 3자 동시 만족 설계의 핵심 원칙

```
원칙 1: "비용 절감은 효율로, 이익은 모두에게"
  - 플랫폼이 중간에서 큰 마진을 취하지 않음
  - 기술 효율화(AI 배차, 묶음배달)로 절감된 비용을 3자가 나눔

원칙 2: "투명성이 신뢰를 만든다"
  - 수수료 구조, 배차 알고리즘, 정산 과정 모두 공개
  - 점주: 정확히 얼마를 왜 내는지 안다
  - 라이더: 콜이 왜 나에게 왔는지 안다
  - 소비자: 배달비가 어디로 가는지 안다

원칙 3: "선택권을 준다"
  - 점주: 월정액 티어 선택, 광고 참여 여부 선택
  - 라이더: 묶음배달 수락 여부 선택, 근무 시간 완전 자유
  - 소비자: 구독 여부 선택, 배달비/픽업 선택

원칙 4: "공급자를 파트너로 대우한다"
  - 점주 자문단, 라이더 운영위원회 → 정책 결정에 참여
  - 일방적 정책 변경 금지. 변경 시 30일 사전 공지 + 의견 수렴
  - 점주/라이더 전용 커뮤니티 운영
```
