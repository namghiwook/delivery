# 소상공인 음식점 직접주문 SaaS 플랫폼 설계서

> "배민/쿠팡이츠 없이 직접 주문을 받을 수 있게 해주는 올인원 SaaS"

---

## 1. 현재 점주가 직접 주문을 받는 경로와 장벽

### 1-1. 네이버 주문 (네이버 스마트플레이스)

**작동 방식:**
- 네이버가 2019년 9월 시작한 음식점/카페 전용 주문결제 서비스
- 소비자가 네이버 지도/검색에서 가게를 찾고 → 메뉴 선택 → 네이버페이로 결제
- QR코드를 통한 매장 내 주문 또는 네이버 지도 앱에서 픽업 주문 가능
- 주문 정보가 스마트플레이스 앱으로 점주에게 전달

**수수료 구조 (2025년 기준):**
- 총 수수료: 매출액의 약 1.1%
  - 네이버가 매출액의 1.1%를 먼저 공제
  - 이 중 0.8%는 신용카드사에 대납
  - 나머지 0.3%가 네이버의 실질적 운영 수수료
- **배민 대비 약 1/7 수준** (배민 7.8% vs 네이버 1.1%)

**핵심 한계:**
- **배달 미연동**: 네이버 주문은 '주문 인입'까지만 가능. 배달대행사와의 연동이 없음
- 대형 프랜차이즈의 자체 배달 플랫폼이 있어야만 매끄러운 배달 서비스 가능
- N배달(가칭) 서비스를 계획했으나, 네이버 CEO가 2024년 "배달 시장 진출 시 어떤 가치를 제공할 수 있을지 아직 답을 찾지 못했다"고 발언
- 점주가 메뉴판/영업시간 업데이트를 안 하는 경우가 많음
- 실질적으로 **포장(픽업) 전용** 서비스에 가까움

### 1-2. 카카오톡 주문하기

**작동 방식:**
- 카카오톡 내에서 음식 주문 가능한 중개 플랫폼
- 치킨, 피자, 중식, 한식 등 9개 카테고리
- 씨엔티테크의 통합형 POS 연동 시스템으로 주문 내역을 가맹점에 전달

**한계:**
- 대형 프랜차이즈 중심 (동네 소상공인 입점이 쉽지 않음)
- 배달 연동은 프랜차이즈 자체 배달망 또는 배달대행사 계약 필요
- 소상공인 독립 가맹점의 개별 입점 절차가 까다로움
- 채널 메시지 발송 시간 제한 (오전 8시~오후 9시)

### 1-3. 인스타그램/블로그를 통한 주문

**현실:**
- DM이나 전화로 주문 → 수동 처리 → 결제 수단 불안정
- 결제: 계좌이체 또는 현장 결제뿐
- 주문 관리 시스템 부재 (엑셀이나 수기로 관리)
- 배달 연동 불가능
- 검색 노출이 알고리즘에 좌우되어 불안정

### 1-4. 점주가 직접 주문을 안 하는 이유

| 장벽 | 상세 |
|------|------|
| **기술적 복잡성** | 네이버 스마트플레이스 설정 + 배달대행 계약 + 결제 시스템 + 주문관리를 각각 따로 해야 함 |
| **배달 연동 불가** | 네이버 주문으로 받은 주문을 배달하려면 배달대행사에 전화/앱으로 별도 접수해야 함 |
| **시간 부족** | 점주는 요리와 매장 관리에 바쁨. IT 설정에 투자할 시간이 없음 |
| **인지 부족** | 네이버 주문의 수수료 이점을 모르는 점주가 많음 |
| **고객 유입 경로** | 소비자가 배민에서 검색하지, 네이버 지도에서 배달 음식을 검색하는 습관이 없음 |
| **신뢰도** | 배민은 리뷰/평점/사진이 풍부. 직접 주문 페이지는 정보 부족으로 신뢰도 낮음 |

## 2. 배달대행사 직접 연동의 현실

### 2-1. 주요 배달대행사 구조

**바로고 (Barogo):**
- 프로그램을 이용하는 **개별 지역배달대행사**들이 식당과 직접 가맹점 계약
- 바로고 본사는 배달수수료에 직접 관여하지 않음
- 지역 대리점 구조 → 지역마다 요금이 다름

**부릉 (VROONG, 메쉬코리아):**
- 본사가 직접 배달수수료를 정하고 가맹점과 직접 계약
- 각 부릉 지사에 배달대행업을 일임
- 상대적으로 체계적인 가격 구조

### 2-2. 배민 없이 배달대행사만 쓸 수 있는가?

**결론: 가능하지만, 매우 번거롭다.**

- 배달대행사는 원래 배달앱과 별개의 서비스. 점주가 직접 계약 가능
- 문제는 **주문 접수 시스템이 없다**는 것
  - 배달대행사는 "어디서 어디로 배달해달라"는 요청만 처리
  - 주문 접수 → 결제 → 조리 → 배달 콜 → 배달 완료의 전체 흐름을 점주가 수동으로 관리해야 함
- 2023년 배달 플랫폼 표준 API 규약이 수립됨 (바로고, 부릉 등 13개사 참여)
  - 이 표준 API를 활용하면 독립적인 주문 시스템에서 배달대행사와 연동 가능

### 2-3. 배달대행 직접 계약 시 건당 비용

| 항목 | 비용 |
|------|------|
| **기본 배달료 (수도권)** | 3,500~4,500원/건 |
| **기본 배달료 (지방)** | 3,000~3,500원/건 |
| **거리 추가 요금** | 거리에 따라 500~2,000원 추가 |
| **피크타임 할증** | 1,000~2,000원 추가 |
| **정부 인정 직접배달 단가** | 5,000원/건 |

### 2-4. 배민 경유 vs 직접 계약 비용 비교

**배민1플러스를 통한 배달 시 (2만 5천원 주문 기준):**

| 비용 항목 | 금액 | 비율 |
|-----------|------|------|
| 중개수수료 (7.8%) | 1,950원 | 7.8% |
| 배달비 (배민라이더) | 3,300원 | 13.2% |
| 결제수수료 | 약 825원 | 3.3% |
| 광고비 (10회 노출당 1주문) | 3,000원 | 12.0% |
| 포장비 | 1,000원 | 4.0% |
| 고객할인 비용 | 825원 | 3.3% |
| **총 공제액** | **약 10,900원** | **약 43.6%** |
| **실제 정산액** | **약 14,100원** | **약 56.4%** |

> 서울시 2025년 12월 발표 '배달플랫폼 상생지수': 입점업체 매출 대비 총 이용 수수료 **16.9~29.3%** (광고비 제외 시). 광고비 포함 시 최대 **30% 이상**.

**직접 주문 + 배달대행 직접 계약 시 (2만 5천원 주문 기준):**

| 비용 항목 | 금액 | 비율 |
|-----------|------|------|
| 네이버 주문 수수료 (1.1%) | 275원 | 1.1% |
| 배달대행 직접 계약 배달비 | 3,500원 | 14.0% |
| 포장비 | 1,000원 | 4.0% |
| **총 공제액** | **약 4,775원** | **약 19.1%** |
| **실제 정산액** | **약 20,225원** | **약 80.9%** |

> **차이: 점주 수취액이 약 6,125원 (24.5%p) 더 많음**

## 3. 이미 이런 서비스를 하는 곳이 있는지

### 3-1. 한국 내 유사 서비스

| 서비스 | 유형 | 현황 | 한계 |
|--------|------|------|------|
| **땡겨요** (신한은행) | 공공배달앱 | 2% 고정 수수료, 광고비 무료. 2025년 회원 697만명, 가맹점 28만개. 서울배달+ 운영사 선정 | 결국 '또 하나의 배달앱'. 점주 독립이 아닌 플랫폼 이동일 뿐 |
| **먹깨비** | 공공배달앱 | 전국 최초 민관 협력 공공배달앱. 지역화폐 연동 | 지역 한정, 사용자 수 부족 |
| **포스피드 (POSFeed)** | 주문통합관리 | 배달앱 주문을 하나의 POS로 통합 관리 | 직접 주문 유도가 아닌 기존 배달앱 통합 관리 도구 |
| **나우웨이팅/테이블링** | 웨이팅/예약 | 매장 대기 관리 특화. 테이블링은 MAU 115만, 회원 300만 | 배달이 아닌 매장 방문 중심 |
| **배달천재** | 배달대행 연동 | 모든 배달대행 플랫폼과 원터치 연동 | 주문 접수가 아닌 배달 콜 관리 도구 |

**핵심 발견: 한국에는 "주문 접수 + 결제 + 배달대행 연동"을 하나로 묶어 소상공인에게 제공하는 서비스가 사실상 존재하지 않는다.**

### 3-2. 해외 유사 서비스 (핵심 벤치마크)

#### ChowNow (미국)
- **모델**: 레스토랑 직접 주문 전문 SaaS. "Commission-Free Online Ordering"
- **가격**:
  - Hub 플랜: $119/월
  - Pro 플랜: $229/월
  - Premier 플랜: $328/월
  - 셋업비: $119~$499 (1회)
  - 결제 수수료: 2.95% + $0.29/건
  - **주문 건당 커미션: 0%** (무제한 주문)
- **작동 방식**:
  1. 레스토랑 자체 웹사이트/브랜드 앱에 ChowNow 주문 시스템 임베드
  2. 소비자가 주문 → POS로 바로 전송 (수동 입력 불필요)
  3. 배달: Flex Delivery로 외부 배달 파트너 자동 디스패치
  4. 마케팅: 자동 이메일 마케팅, 로열티 프로그램 내장
- **핵심**: Toast, Square, Revel, Clover 등 주요 POS와 통합

#### Toast (미국)
- **모델**: 레스토랑 올인원 POS + 온라인 주문
- **온라인 주문 모듈**: 월 $75 추가 또는 건당 약 3% 커미션
- **결제 수수료**: 2.49% + $0.15/건 (대면), 3.50% + $0.15/건 (비대면)
- 전체 월 비용: $415~$484 (POS + 온라인주문 + 필수 애드온)
- 배달: DoorDash Drive 등 외부 배달 파트너 연동

#### Square Online (미국)
- **모델**: 무료 온라인 주문 페이지 + POS 통합
- **수수료**: 2.9% + $0.30/건 (온라인 결제)
- **월 비용**: 기본 무료 (프리미엄 플랜 있음)
- POS와 자동 동기화

### 3-3. 왜 한국에서는 이 모델이 활성화되지 않았는가?

| 이유 | 설명 |
|------|------|
| **배민의 압도적 점유율** | 배민 57.6% + 쿠팡이츠 35.3% = 93% 독과점. 소비자가 "배달=배민" 인식 |
| **소비자 행동 패턴** | "뭐 먹지?" → 배민 앱 열기. 개별 가게 사이트를 직접 찾아가는 습관이 없음 |
| **배달대행 API 접근성** | 바로고/부릉의 API가 대형 플랫폼 위주로 연동. 소규모 SaaS가 연동하기 어려움 |
| **POS 파편화** | 한국 음식점 POS 시스템이 매우 다양하고 표준화되지 않음 |
| **초기 투자 대비 불확실한 ROI** | 점주 입장에서 배민을 끊고 새 시스템을 도입하는 리스크가 큼 |
| **미국과의 구조적 차이** | 미국은 Google Maps/Yelp에서 가게를 찾고 자체 사이트로 이동하는 문화. 한국은 배달앱이 검색~주문~결제~배달을 일괄 처리 |

## 4. 구체적 사업 모델 설계: "직주(직접주문)" 플랫폼

### 4-1. 서비스명(가칭): **내가게주문** (또는 "직주", "마이오더")

> 핵심 가치 제안: "배민 수수료의 1/5로 직접 주문받으세요. 설정은 저희가 다 해드립니다."

### 4-2. 점주 가입 후 Step by Step

```
[STEP 1] 가입 및 기본 정보 입력 (10분)
├── 사업자등록번호 입력 → 자동으로 사업자 정보 조회
├── 가게명, 주소, 전화번호, 영업시간
├── 메뉴 등록 (사진 촬영 가이드 + AI 메뉴판 인식)
└── 은행 계좌 연결 (정산용)

[STEP 2] 주문 페이지 자동 생성 (즉시)
├── 가게 전용 주문 웹페이지 자동 생성 (예: myorder.kr/맛있는치킨)
├── 네이버 스마트플레이스 연동 설정 (가이드 제공)
├── QR코드 자동 생성 (매장 내 부착용, 전단지용)
├── 카카오톡 채널 연동 (주문 링크 자동 배포)
└── 인스타그램 프로필 링크용 URL 제공

[STEP 3] 배달대행 연동 (1영업일)
├── 바로고/부릉 등 배달대행사 자동 매칭 (지역 기반)
├── 대행사 계약 대행 (플랫폼이 중간에서 일괄 계약)
├── 배달비 자동 산정 (거리/시간 기반)
└── 원클릭 배달 콜 시스템 연동

[STEP 4] 결제 시스템 연동 (자동)
├── 네이버페이 (수수료 1.1%)
├── 카카오페이
├── 신용카드 (PG사 연동)
└── 토스페이

[STEP 5] 운영 시작
├── 태블릿/스마트폰 앱으로 주문 접수
├── 주문 알림 (소리 + 푸시)
├── 조리 완료 → 원터치 배달 콜
├── 자동 정산 (D+1~D+3)
└── 매출/주문 통계 대시보드
```

### 4-3. 소비자 주문 흐름

```
[경로 1: 네이버 검색]
네이버에서 "OO동 치킨" 검색
→ 스마트플레이스에 가게 노출
→ "주문하기" 버튼 클릭
→ 내가게주문 주문 페이지로 이동
→ 메뉴 선택 → 배달/포장 선택
→ 네이버페이/카카오페이/카드 결제
→ 주문 완료 → 실시간 조리/배달 상태 확인

[경로 2: QR코드 / 직접 링크]
전단지/매장 내 QR코드 스캔
→ 내가게주문 주문 페이지
→ (이하 동일)

[경로 3: 카카오톡 채널]
가게 카카오톡 채널에서 "주문하기" 클릭
```

## 6. 이 모델의 근본적 한계와 대응 전략

### 6-1. 핵심 딜레마: "집약의 힘" vs "분산의 자유"

**배민의 핵심 가치 = 원스톱 발견(Discovery)**
- 소비자: "뭐 먹지?" → 배민 열기 → 수십 개 가게 비교 → 주문
- 이 흐름에서 개별 가게 주문 페이지가 끼어들 여지가 없음

**직주 플랫폼의 딜레마:**
- 소비자가 이미 그 가게를 알고 있어야만 직접 주문이 발생
- "발견(Discovery)"이 아닌 "재주문(Retention)"에 최적화된 모델

### 6-2. 한계와 대응 전략

| 한계 | 심각도 | 대응 전략 |
|------|--------|-----------|
| **소비자가 배민 하나로 모든 가게를 검색하는 편의성을 포기할 이유가 없다** | ★★★★★ | 이 서비스의 타겟은 "신규 고객 획득"이 아니라 **"기존 고객의 재주문 전환"**. 한 번 방문한 고객을 직접 주문 채널로 유도 |
| **각 가게마다 다른 주문 페이지에 들어가야 하는 불편함** | ★★★★☆ | 네이버 지도/카카오맵이라는 기존 습관 활용. 새 앱 설치 불필요. 웹 기반 |
| **배달비가 소비자에게 더 비쌀 수 있음** | ★★★☆☆ | 점주가 절감한 수수료를 "직접주문 할인"으로 환원. "배민에서 시키면 16,000원, 직접 주문하면 14,000원" |
| **배달대행사 API 연동의 기술적 난이도** | ★★★☆☆ | 배달 플랫폼 표준 API(2023) 활용. 바로고/부릉과 B2B 제휴 |
| **점주의 디지털 리터러시** | ★★★★☆ | "화이트글러브 온보딩" = 직원이 직접 방문하여 설정. 태블릿 제공 |
| **결제 신뢰도** | ★★☆☆☆ | 네이버페이/카카오페이 등 이미 신뢰받는 결제 수단 활용 |

### 6-3. 실현 가능한 타겟 시장

**이 플랫폼이 작동하는 가게 유형:**

1. **단골 비율이 높은 동네 음식점** (분식, 중국집, 치킨집 등)
   - "매번 배민으로 시키던 단골"을 직접 주문으로 전환
   - QR코드 전단지 + "직접 주문 시 2,000원 할인"

2. **SNS 팔로워가 많은 맛집**
   - 인스타 팔로워 → 직접 주문 페이지로 유도
   - 배민 수수료를 아끼고 그 비용으로 더 좋은 재료 사용

3. **포장/픽업 비중이 높은 가게**
   - 배달이 필요 없으면 배달대행 연동도 불필요
   - 네이버 주문 수수료 1.1%만으로 운영 가능
   - 배민 포장 수수료 6.8% → 직주 1.1%로 전환 (6배 절감)

4. **프랜차이즈가 아닌 개인 음식점**
   - 프랜차이즈는 본사 시스템이 있지만, 개인 음식점은 없음
   - 이 빈틈을 채우는 것이 핵심

### 6-4. 성장 전략: "포장부터 시작, 배달로 확장"

```
[Phase 1: 포장 주문 전환] (0~6개월)
├── 배민 포장 수수료 6.8%에 반발하는 점주 확보
├── "포장 주문만" 직주로 전환 (배달은 계속 배민 사용)
├── 점주 부담: 네이버 1.1% vs 배민 6.8% → 즉각적 비용 절감
└── 소비자 행동 변화 최소화 (매장 앞에서 QR 스캔)

[Phase 2: 단골 배달 전환] (6~12개월)
├── 포장 고객 중 배달 수요 파악
├── "이 가게에서 직접 주문하면 2,000원 할인" 프로모션
├── 배달대행 연동 시작 (바로고/부릉)
└── 카카오 알림톡으로 재주문 유도

[Phase 3: 에코시스템 구축] (12~24개월)
├── 동네 단위 "직주 가맹점 모음" 페이지 (지역 마켓플레이스)
├── 네이버 지도 내 직주 가맹점 배지 표시
├── 소비자 포인트/쿠폰 시스템
└── 점주 간 공동 프로모션 (예: "OO동 직주 축제")
```

### 6-5. 왜 "지금"이 적기인가?

| 트리거 | 설명 |
|--------|------|
| **배민 포장 수수료 6.8% 신설 (2025.4.14)** | 점주 반발 극심. "포장도 수수료?"라는 분노가 대안 탐색의 동기 |
| **서울시 배달플랫폼 상생지수 발표 (2025.12)** | 수수료 29.3%라는 수치가 공론화. 규제 가능성 |
| **배달 플랫폼 표준 API 수립 (2023)** | 기술적으로 배달대행사 연동이 가능해진 시점 |
| **네이버 주문 인프라 존재** | 수수료 1.1%의 결제 인프라가 이미 있음 |
| **배달시장 40조 돌파 (2025)** | 시장이 충분히 크고, 수수료 불만도 충분히 큼 |
| **땡겨요의 성공이 증명한 것** | 저수수료 대안에 대한 수요 존재 (697만 회원, 28만 가맹점) |

## 9. 결론

### 이 사업의 본질

이 플랫폼은 **"배민 킬러"가 아니다**. 배민은 "발견(Discovery)" 플랫폼이고, 이 서비스는 **"재주문(Retention)" 인프라**다.

- 신규 고객은 배민/쿠팡이츠/네이버에서 발견
- 한 번 방문한 고객은 직접 주문 채널로 전환
- 전환된 고객의 재주문에서 수수료를 7.8% → 1.5%로 줄임
- **점주가 고객 데이터를 소유**하게 됨 (배민에서는 불가능)

### 한 줄 요약

> **"배민으로 첫 손님을 받고, 직주로 단골을 관리하라"**
>
> — 점주의 배민 의존도를 100%에서 50%로 낮추는 것만으로도, 월 500만원 매출 기준 연 400~800만원을 절감할 수 있다.
